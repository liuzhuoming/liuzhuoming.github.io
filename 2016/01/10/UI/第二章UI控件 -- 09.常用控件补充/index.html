09.常用控件补充

第二章UI控件 – 09.常用控件补充

一、学习目标

  • UISegmentedControl 介绍和使用
  • UIActivityIndicatorView的代理事件
  • UIProgressView 介绍和使用
  • UIStepper 介绍和使用

二、课程内容

至今我们学习了不少的控件：UIButton、UILabel、UIImageView和UITextField等，希望大家能掌握控件的基本学习方法。

1.UISegmentedControl 介绍和使用

分段控制器，常用于执行多选一的情景。

接下我们看下如何用代码来设置UISegmentedControl的样式。

　通过查看UISegmentedControl.h可以发现UISegmentedControl像UIButton一样继承于UIControl,所以它是可以用户交互的。

2.UIActivityIndicatorView 介绍和使用

活动指示器通常用于在App做耗时操作是提示用户App当前还在处于活动状态。比如在网络请求时，使用活动提示器可以避免长时间不更新页面，被用户误以为App卡死从而强行关闭App。

活动指示器的常用属性和方法如下
| 常用属性 | 定义 |
| ——– | :—– : |
| 指示器的样式 | activityIndicatorViewStyle |
| 当停止动画的时候，是否隐藏。默认为YES |hidesWhenStopped |
| 实例化指示器对象，根据样式设置尺寸，不需要手动设置| -(id)initWithActivityIndicatorStyle:(UIActivityIndicatorViewStyle)style; |
| 开启动画 | - (void)startAnimating; |
| 关闭动画 | - (void)stopAnimating; |
| 是否在动画 | - (BOOL)isAnimating;|

3.UIProgressView 介绍和使用

进度条与UISlider不同，他不可以与用户进行交互。

常用属性如下：
| 常用属性 | 定义 |
| ——– | —– |
| progressViewStyle | 设置进度条风格样式 |
| progress | 设置进度条进度(0.0-1.0之间，默认为0.0) |
| progressTintColor | 设置已走过进度的进度条颜色 |
| trackTintColor | 设置未走过进度的进度条颜色 |

4.UIStepper 介绍和使用

这是一个用于递增的控件，继承自UIControl，它主要的事件是UIControlEventValueChanged。

常用属性如下
| 常用属性 | 定义 |
| ——– | —– |
| value | 当前所表示的值，默认0.0 |
| minimumValue | 最小可以表示的值，默认0.0 |
| maximumValue | 最大可以表示的值，默认100.0 |
| stepValue | 每次递增或递减的值，默认1.0 |
| continuous | 控制是否持续触发UIControlEventValueChanged事件。默认YES，即当按住时每次值改变都触发一次UIControlEventValueChanged事件，否则只有在释放按钮时触发UIControlEventValueChanged事件。 |
| autorepeat | 控制是否在按住是自动持续递增或递减。默认YES。 |
| wraps | 控制值是否在[minimumValue,maximumValue]区间内循环。默认NO。 |
