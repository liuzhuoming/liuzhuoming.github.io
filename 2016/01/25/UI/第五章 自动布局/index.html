自动布局介绍

从iOS6/xcode4开始，苹果开始提供了Autolayout——一种对不同屏幕尺寸有更好兼容的自动布局机制。但是由于当时的屏幕比较少，而且当时的Xcode4并没有很好的支持。但是随着iPhone6和iPhone6plus的发布，Xcode对Autolayout的支持，大家开始慢慢放弃frame或Autosizing布局。
苹果官方对其描述是：AutoLayout是一种基于约束的，描述性的布局系统。 Auto Layout Is a Constraint-Based, Descriptive Layout System.

  • 参照 frame布局下，控件参考的都是父视图，Autolayout可以参考父视图和同级的兄弟视图。frame.size是固定的，以致于常常需要为不同屏幕尺寸计算各自的位置点坐标和大小了。Autolayout则可以根据屏幕或需求进行自动的拉伸压缩。
  • 约束 和以往定义frame的位置和尺寸不同，AutoLayout的位置确定是以所谓相对位置的约束来定义的，比如x坐标为superView的中心，y坐标为屏幕底部上方10像素等。

代码的应用

NSLayoutConstraint简单介绍

Autolayout同样可以用代码(NSLayoutConstraint)来实现，其主要思想是firstItem.firstAttribute {==,<=,>=} secondItem.secondAttribute * multiplier + constant。

VFL简单介绍

NSLayoutConstraint实在是太麻烦，连苹果自己都看不下去了，推出了VFL。VFL全称是Visual Format Language,翻译过来是“可视化格式语言”
@"H:|-10-[view]-20-|"意思就是view距离superview的左边10点，右边20点，这样是不是就可以确定了这个view的宽度了？
其中H:表示横向，|表示父视图边缘，-10-表示10点距离，[view]表示子视图。

Masonry

不要被上面冗长的NSLayoutConstraint，和奇怪的VFL给吓到，我们现在来重点学习一个名为Masonry的框架，它是一个轻量级的布局框架，拥有自己的描述语法，采用更优雅的链式语法封装自动布局 简洁明了，并具有高可读性。

Masonry NSAutoLayout 说明
left NSLayoutAttributeLeft 左侧
top NSLayoutAttributeTop 上侧
right NSLayoutAttributeRight 右侧
bottom NSLayoutAttributeBottom 下侧
leading NSLayoutAttributeLeading 首部
trailing NSLayoutAttributeTrailing 尾部
width NSLayoutAttributeWidth 宽
height NSLayoutAttributeHeight 高
centerX NSLayoutAttributeCenterX 横向中点
centerY NSLayoutAttributeCenterY 纵向中点
baseline NSLayoutAttributeBaseline 文本基线

对中文来说 left = mas_left = leading
left、right 不能与 leading、trailing混用
